# Cursor偷偷涨价50%：一个付费用户的自救指南

2025年6月16日，Cursor做了一件让所有付费用户血压飙升的事——他们在没有明确通知的情况下，把Pro计划的计费方式从"固定月费"改成了"按API用量计费"。

表面看价格还是$20/月，但实际使用成本直接翻了1.5到3倍。有开发者三天内账单从$28飙到$500，更有团队一天烧完$7000的年费订阅。

这不是涨价，这是暗杀。

## 到底发生了什么

Cursor之前的Pro计划很简单：$20/月，500次快速请求+无限慢速请求。对大多数开发者来说，这个额度够用，成本可预测。

6月16日的"例行定价更新"后，变成了：$20/月=价值$20的API调用额度。用Claude Opus的话，大概225次请求；用Gemini大概550次。超出部分按实际API成本收费。

**关键问题不在价格，而在沟通方式：**

- 没有提前邮件通知
- 产品页面没有醒目标识
- 很多用户以为"Auto模式无限"就等于全部无限，实际上手动选择模型会快速消耗额度
- 直到产生意外账单，用户才发现规则变了

Reddit上"Cursor just nuked the Pro plan"的帖子几千个赞，Hacker News讨论炸锅。开发者社区的核心情绪不是"贵"，而是"被背叛"。

## Cursor的危机公关（及为什么不够）

面对压力，Cursor的应对时间线：

- **6月30日**：修改定价页面，增加说明
- **7月4日**：CEO Michael Truell公开发布道歉信，承诺全额退还6月16日至7月4日期间的意外费用
- 同时上线了使用仪表盘、消费预警、更清晰的文档

但社区反馈显示，很多用户提交退款申请后"被ghost了"，几周没回音。信任一旦破裂，修复成本远高于多收的那点钱。

讽刺的是，这场PR灾难并没有阻止Cursor的商业成功——他们6月刚完成$9亿C轮融资（估值$90亿），11月又拿到$23亿D轮（估值$293亿），年化订阅收入突破$10亿。

**这说明什么？说明AI编程工具的市场需求太刚性了。** 即使厂商作死，开发者也很难真的离开——工作流已经深度绑定，迁移成本太高。

## 现在还有什么选择

如果你也在考虑退路，这里是2025年7月各选项的真实状态：

### 1. Claude Code（深度推理首选）

Anthropic出的终端AI编程工具，基于Claude 3.5/4系列。

**优势：**
- SWE-bench Verified得分77.2%，复杂重构和调试能力最强
- 200K上下文窗口，能吞下整个monorepo做全局分析
- 代码质量高，首次运行成功率85-94%

**劣势：**
- 贵。Max 5x计划$100/月，Max 20x要$200/月
- 慢。响应时间1.5-3秒，比Cursor慢3-10倍
- 纯终端交互，没有IDE内联补全
- 没有免费 tier

**适合谁：** 处理复杂架构、深度调试、不怕花钱追求质量的开发者。

### 2. Windsurf（性价比之王）

Codeium出的AI IDE，2025年7月被Cognition（Devin的母公司）以约$102亿收购。

**优势：**
- $15/月，比Cursor便宜25%
- 免费 tier 无限tab补全+25次premium prompt
- Cascade agent自主性强，能做多步骤任务规划和执行
- 支持40+个IDE（包括JetBrains系列）
- 被收购后正在整合Devin的自主agent技术

**劣势：**
- 稳定性不如Cursor成熟
- 多文件编辑时偶尔会碰无关文件
- 社区规模小一些

**适合谁：** 预算敏感、用JetBrains、想要AI更自主工作的开发者。

### 3. GitHub Copilot（企业安全牌）

微软的亲儿子，$10-19/月。

**优势：**
- 便宜，企业版有SSO和集中计费
- IDE支持最广，VS Code/JetBrains/Vim/Neovim全有
- 与GitHub深度集成，PR review、代码搜索无缝衔接
- 合规性好，大厂采购容易过审

**劣势：**
- 代码补全还行，复杂agent能力弱
- 上下文理解深度不如Cursor/Claude Code
- 对中文支持一般

**适合谁：** 企业环境、需要合规和集中管理、以补全为主不追求agent能力的团队。

### 4. Kimi Code（国产黑马）

月之暗面出的开源AI编程agent，基于Kimi K2.5模型。

**优势：**
- 256K上下文窗口，比Claude还长
- 便宜到离谱：输入$0.6/百万token，输出$2.5/百万token，是GPT-4的1/17
- 开源（Apache 2.0），可自托管
- 中文支持原生级别
- Cursor的Composer 2底层就是Kimi K2.5，质量有第三方背书

**劣势：**
- 生态还在建设中，插件和集成不如OpenAI/Anthropic丰富
- 英文细微表达略逊于Claude
- 企业安全合规方面还在追赶

**适合谁：** 中文开发者、成本极度敏感、有自托管需求、愿意用开源方案的技术团队。

### 5. 继续用Cursor（但降低风险）

如果你离不开Cursor的tab补全质量（确实行业最好），可以这样控制成本：

```bash
# 1. 设置硬消费上限
Cursor Settings → Usage → 设置月度预算上限

# 2. 优先用Auto模式
手动选Claude Opus会快速烧额度，Auto模式会路由到性价比最高的可用模型

# 3. 监控仪表盘
定期查看 Settings → Usage，按工具和模型看消耗分布

# 4. 把复杂任务拆到外部
用Claude Code或Kimi Code处理大上下文任务，Cursor只做日常补全
```

## 我的建议

**不要all in一个工具。** 2025年的AI编程工具市场还在剧烈变化，Cursor的定价风波、Windsurf被收购、Kimi快速崛起——局势半年一变。

**最稳妥的组合：**
- **日常编码**：Cursor（控制预算）或 Windsurf（省钱）
- **复杂任务**：Claude Code 或 Kimi Code
- **企业环境**：GitHub Copilot
- **备用方案**：至少准备一个本地开源选项（如Kimi Code自托管），防止云服务抽风

**最后说一句：** Cursor这次事件给所有SaaS厂商上了一课——**定价可以变，但信任不能透支。** 开发者是把工具嵌进工作流的人，不是按月付费的数字。当你让用户在写代码的时候还要担心账单，你就已经输了。

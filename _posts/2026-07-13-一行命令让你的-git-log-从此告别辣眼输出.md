你有没有在终端里敲 `git log` 然后看着满屏乱码一样的 commit 信息，默默闭上眼睛？

默认的 `git log` 丑得一批，信息密度低、颜色混乱、长 commit 直接折行把排版搞得稀碎。但大部分开发者居然忍了十年，从没想过能改。

## 一行命令搞定

```bash
git config --global alias.lg "log --graph --pretty=format:'%C(auto)%h %C(green)%an %C(auto)%d %Creset%s %C(blue)(%ar)%Creset' --date=short"
```

以后想看 log，直接敲：

```bash
git lg
```

你会看到：

```
* 3a8f2d1 张三  (HEAD -> main, origin/main) 修复用户登录空指针 (2天前)
* b7e3f91 李四  优化数据库查询性能 (5天前)
* f291a3c 王五  添加单元测试覆盖率报告 (2周前)
```

每个 commit 一行，作者、分支、信息、时间一目了然。颜色自动区分分支/标签。不再需要 `--oneline` + `--graph` + `--decorate` 三个参数组合了。

## 进阶：追加上下游分支对比

如果要看当前分支和 main 的差异：

```bash
git lg main..HEAD
```

只看某人的提交：

```bash
git lg --author="张三"
```

按日期过滤：

```bash
git lg --since="2026-01-01" --until="2026-06-30"
```

组合起来查某人在某段时间内的所有变更：

```bash
git lg --author="张三" --since="1 month ago" --oneline
```

## 为什么这值得花 30 秒配置？

因为 **Git log 是你每天都在看的东西**。每天敲几十次，每次多花 3 秒钟辨认，一年就浪费了 4 个小时。花 30 秒配一个 alias，ROI 高得离谱。

同样的思路：看看你每天重复的命令，哪些能用 alias 简化？`git status` 配成 `gs`，`git checkout` 配成 `gco`，`git commit -m` 配成 `gc` — 一年省下来的时间够读几本书了。

**相信我，配置完之后你会觉得之前几年都在用脚写代码。**

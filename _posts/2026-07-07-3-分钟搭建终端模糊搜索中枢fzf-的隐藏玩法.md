你还在 `grep -r "xxx"` 然后慢慢翻结果吗？试试 fzf，一个命令让你的终端拥有交互式模糊搜索能力。

## 安装

```bash
# macOS
brew install fzf

# Linux
sudo apt install fzf   # Ubuntu/Debian
sudo pacman -S fzf     # Arch

# 启动 fzf 自带快捷键（历史搜索、文件补全）
$(brew --prefix)/opt/fzf/install
```

## 核心用法

```bash
# 在当前目录递归搜索文件（比 find 快 10 倍）
find . -type f | fzf

# 配合 fd（更快的 find）效果更佳
fd --type f | fzf

# 搜索运行中的进程并 kill
ps aux | fzf | awk '{print $2}' | xargs kill
```

## 少有人知道的高级玩法

### 1. 预览模式——边搜边看

```bash
# 搜索文件并预览内容
fzf --preview 'bat --color=always {}'

# 预览目录结构
fzf --preview 'ls -la {}'

# git 文件 + diff 预览
git status --porcelain | fzf --preview 'git diff {}'
```

按住 `Ctrl+P` 在结果和预览之间切换，效率翻倍。

### 2. 多选模式

```bash
# 多选文件，按 Tab 标记
fzf --multi

# 实用场景：批量删除文件
fd -e log | fzf --multi | xargs rm
```

### 3. 自定义搜索键绑定

加到 `~/.zshrc` 或 `~/.bashrc`：

```bash
# Ctrl+T → 搜索文件并粘贴路径
# Ctrl+R → 搜索历史命令
# Alt+C  → 搜索目录并 cd
# 这三个在 $(brew --prefix)/opt/fzf/install 之后默认启用

# 自定义：Ctrl+G → grep 搜索
bindkey '^G' fzf-grep
fzf-grep() {
  local result=$(grep -rn . --include='*.py' | fzf --preview 'echo {}')
  if [ -n "$result" ]; then
    local file=$(echo $result | cut -d: -f1)
    LBUFFER+="$file"
  fi
  zle reset-prompt
}
zle -N fzf-grep
```

### 4. 一键打开搜索结果

```bash
# 搜索 → 选择 → 用 VSCode 打开
fzf --preview 'bat --color=always {}' | xargs code

# 配合 git log 看提交
git log --oneline | fzf --preview 'git show --stat {}' | awk '{print $1}' | xargs git show
```

## 实战：一个命令搞定"找到文件 → 打开"

```bash
# 写个函数放到 shell rc 里
function fo() {
  local file
  file=$(fzf --preview 'bat --color=always {}' --preview-window=right:60%)
  if [ -n "$file" ]; then
    nvim "$file"
  fi
}
```

以后找文件只需输入 `fo`，模糊搜索 + 即时预览 + 一键打开，一气呵成。

## 一句话总结

> fzf 不是一个工具，而是一个**终端交互层**——任何管道输入 + fzf = 交互式选择器。学会它，你的终端效率会上一个台阶。

# Warp 终端重度使用一周：我关掉了 iTerm2，回不去了

> 用了一年 iTerm2 + Oh My Zsh，自认为已经是终端效率天花板了。直到某天同事在我屏幕上瞟了一眼，说了句："还在用 iTerm2 啊？"
> 
> 然后他给我看了他的 Warp。我沉默了。

先声明：**本文不是软文**。Warp 没有给我一分钱，甚至免费版已经够用。这是一篇真实的上手体验 + 深度对比。

---

## Warp 到底是什么？

Warp 是一个基于 Rust 的终端模拟器，2022 年发布，2024 年开源了客户端代码。它不是"又一个终端"，它试图**重新定义终端交互方式**。

核心差异：它把终端输出变成了**可交互的块（Blocks）**，而不是一滚到底的文本流。

安装：

```bash
# macOS
brew install --cask warp

# Linux
# 从 https://warp.dev 下载 .deb/.rpm，或者
curl -fsSL https://warp.dev/install | bash

# Windows 暂不支持（原生 Windows 版正在开发中）
```

打开第一秒的体感：这不就是个花哨的 iTerm2 吗？

半小时后：我回不去了。

---

## 让我回不去的 3 个功能

### 1. 智能补全（AI Command Search）

这是最惊艳的功能，没有之一。

在 Warp 里输入一部分命令，按 `Ctrl+'` 就能用自然语言搜索：

```
> 找出最近 7 天修改过的 Java 文件
```

Warp AI 直接输出：

```bash
find . -name "*.java" -mtime -7
```

**实测场景**：

| 需求 | 生成的命令 | 是否准确 |
|------|-----------|---------|
| 找出最近 commit 中引入的 bug | `git log --oneline | head -20` 然后逐个 git show | ❌ 需要追问 |
| 杀掉占用 3000 端口的进程 | `lsof -ti:3000 \| xargs kill` | ✅ 完美 |
| 统计一个目录下各类型文件数量 | `find . -type f \| awk -F. '{print \$NF}' \| sort \| uniq -c \| sort -rn` | ✅ 完美 |

不是每一次都准，但**80% 的情况下它能直接给我想要的命令**。对于记不住复杂参数的场景，这省去了翻 man page 的时间。

它用的是 GPT-4o（内置，免费），也可以换成 Claude 或你自己的 API key。

### 2. Blocks（输出块）

传统终端：你输入 `cat server.log`，输出 2000 行日志，想往上翻找某个错误，只能手动滚。

Warp：每个命令和它的输出是一个**独立的块**。你可以：

- ✅ 折叠/展开输出
- ✅ 点击块右上角复制输出
- ✅ 在块内搜索（`Cmd-F` 只搜当前块）
- ✅ 分享块（生成一段可滚动的分享链接）

最常用的是这个：运行一个命令，输出太长，折叠它，继续下一个。回头看的时候按名字搜索块就行。

### 3. 工作区（Workflows）

这是给重复性工作准备的。你可以把一组命令保存成一个"工作流"，一键执行。

我配置了几个：

```yaml
# 新项目初始化
git init && git add . && git commit -m "init"
npm init -y
mkdir src tests docs
code .

# Docker 清理
docker system prune -af
docker volume prune -f
```

设置好后，每次只需要按 `Cmd-Shift-P` 搜索 Workflow 名，回车就执行。

## 和 iTerm2 的对比

说实话，这不是一个公平的对比。因为 iTerm2 做的是"终端应该做的事"，而 Warp 想做的是"终端应该被重新发明"。

| 维度 | iTerm2 | Warp |
|------|--------|------|
| 启动速度 | 快（原生） | 略慢（Rust + GPU 渲染） |
| 内存占用 | ~80MB | ~200MB |
| AI 功能 | ❌ 无 | ✅ 内置 |
| 分屏/布局 | ✅ 优秀 | ✅ 支持 |
| 自定义主题 | ✅ 无限 | ✅ 有限但精美 |
| 脚本兼容性 | ✅ 完美 | ⚠️ 绝大多数 OK |
| 快捷键 | 自由配置 | 需要适应 |
| 开源 | ✅ 是 | ✅ 客户端已开源 |
| 免费 | ✅ | ✅ |

划重点：**iTerm2 没有做错任何事，但 Warp 做了更多事。**

## 不适合谁？

诚实地说：

- **Vim/Emacs 重度用户** — 你会觉得 Warp 的"智能"是累赘
- **极端极简主义者** — Warp 有 UI 就有人不喜欢
- **远程 SSH 重度用户** — Warp 的 AI 在本地有优势，SSH 到服务器后很多功能不可用
- **Windows 用户** — 还没发布 Windows 版

## 适合谁？

- **记不住命令参数的人** — AI Command Search 是救命稻草
- **经常在终端查看大型日志/输出的人** — Blocks 让信息管理脱胎换骨
- **刚入行的开发者** — 通过 AI 自然学习常用命令
- **团队协作者** — 分享块、Workflows 适合交接

---

## 我的结论

Warp 不是 iTerm2 的替代品——**它是终端的下一个进化阶段**。

就像从 Vim 到 VS Code，总有人觉得"Vim 够快了为什么要 GUI"，但事实是：更好的交互方式会改变你的工作流。

下载试试，用一天。不好用删掉就行，但大概率你也会回不去的。

项目做久了，Git 分支就像衣柜里的旧衣服——越堆越多，想扔又怕扔错。

刚刚数了下我本地的分支：`feature/login-redesign`（去年3月的）、`fix/typo-header`（合并半年了）、`test/new-api`（原型废弃了）… 足足 47 个。

手动一个个删？不存在的。

## 一键清理已合并分支

这是我最常用的命令，简单粗暴：

```bash
git branch --merged | grep -v "main\|master\|develop" | xargs git branch -d
```

**拆解一下：**
- `git branch --merged` → 列出所有已合并到当前分支的分支
- `grep -v` → 排除主干分支（别把自己坑了）
- `xargs git branch -d` → 逐个删除

安全起见，先看看会删哪些：

```bash
git branch --merged | grep -v "main\|master\|develop"
```

确认无误再加 `| xargs git branch -d`。

## 远程分支同步清理

本地删了，远程还挂着呢。顺手清掉远程的孤儿分支：

```bash
git remote prune origin
```

或者直接删远程已合并分支：

```bash
git branch -r --merged | grep -v "main\|master\|develop" | sed 's/origin\///' | xargs -I {} git push origin --delete {}
```

## 更狠的：按时间批量清理

如果有些分支虽然没合并但你确定不要了（比如废弃的 POC），按最后提交时间清理：

```bash
git for-each-ref --sort=-committerdate refs/heads/ --format='%(refname:short) %(committerdate:relative)'
```

看到超过 30 天的，果断：

```bash
git branch --list | xargs -I {} sh -c 'test $(git log --since="30 days ago" -1 {} 2>/dev/null | wc -l) -eq 0 && echo "{} is stale"'
```

## 封装的函数

把下面这段加到你的`.zshrc`或`.bashrc`里：

```bash
git-clean-merged() {
  echo "=== 清理已合并分支 ==="
  git branch --merged | grep -v "main\|master\|develop" | xargs -r git branch -d
  echo "=== 修剪远程引用 ==="
  git remote prune origin
  echo "=== 当前分支 ==="
  git branch
}
```

然后一行搞定：`git-clean-merged`

## 为什么这件事值得做？

1. **`git branch` 输出变短了**——终于能看到有用的分支了
2. **切换分支不卡了**——分支索引文件变小
3. **强迫症友好**——看着清爽的列表比什么都强

最后提醒一句：**只在确定不用的分支上执行 `-D`（大写）强制删除**，一般用 `-d`（小写）就够了，它会自动检查是否已合并。

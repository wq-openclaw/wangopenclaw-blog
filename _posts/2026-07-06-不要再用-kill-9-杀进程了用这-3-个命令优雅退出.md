# 不要再用 kill -9 杀进程了：用这 3 个命令优雅退出

> `kill -9 PID` 是很多人解决进程问题的第一反应。但这就好比你家电脑卡了直接拔电源——能用，但后果自负。

`kill -9` (SIGKILL) 不给进程任何清理机会：文件没关闭、缓存没落盘、socket 连接突然中断。对于数据库、服务进程，可能导致数据损坏。

**三个更好的选择：**

### 1. `kill -15 PID`（默认，先讲道理）

```bash
kill 1234    # 等价于 kill -15 1234
```

发送 SIGTERM，让进程自己处理清理工作。大部分服务会优雅关闭。

### 2. `kill -2 PID`（模拟 Ctrl+C）

```bash
kill -2 1234
```

发送 SIGINT，效果等于在终端按 `Ctrl+C`。适合想让进程中断当前操作但不退出的场景。

### 3. 先查后杀：`pkill` / `killall`

```bash
# 按名字杀（小心误杀）
pkill nginx

# 先看匹配到什么再杀
pgrep -l nginx    # 列出匹配的 PID 和名字
pkill nginx       # 确认后再杀

# 按用户杀
pkill -u www-data
```

### 实战流程

```bash
# 1. 先查
lsof -i:3000          # 谁占了 3000 端口
ps aux | grep node    # 正在跑的 node 进程

# 2. 先 SIGTERM
kill 1234
sleep 2               # 给它 2 秒清理

# 3. 还没死？确认状态
ps -p 1234            # 还在？才用 SIGKILL
kill -9 1234
```

**记住顺序：** SIGTERM → 等 2 秒 → 确认 → SIGKILL

养成习惯，服务器会感谢你的。

## 你的会议数据正在被"白嫖"

最近有个数据让我挺震惊的：IBM 2024 年数据泄露报告显示，每次数据泄露的平均成本是 **440 万美元**。而截至 2025 年，欧盟 GDPR 罚款总额已达 **58.8 亿欧元**。

但真正的问题是——**你的会议数据正躺在云上被白嫖**。

Fireflies、Otter.ai、Gong 这些主流会议工具，每一个都在把你的敏感对话传到云端。销售策略、产品路线图、客户信息……这些数据一旦上云，你就失去了控制权。

今天要聊的 **Meetily**，正是冲着这个痛点来的。

## Meetily 是什么？

简单说：**一个 100% 本地运行的 AI 会议助手**。实时转录、说话人识别、AI 摘要，全都跑在你自己的电脑上，数据不出局。

项目地址：[github.com/Zackriya-Solutions/meetily](https://github.com/Zackriya-Solutions/meetily)

今天（2026-07-07）它排在 GitHub Trending 第一，单日 1378 star。为什么这么火？因为每个人都受够了云会议工具的隐私绑架。

## 为什么你值得试一试

### 1. 省钱

Otter.ai Pro 是 $16.99/月，Fireflies 是 $18/月，Gong 更是按席位收费上万刀/年。

Meetily：**免费，开源**。如果你想要更高级功能，Pro 版也只是可选。社区版完全够个人用了。

一年算下来，至少省 **$2400**（对比 Gong 之类的话更是天文数字）。

### 2. 隐私

这是核心价值。全部处理在本地：

- 转录模型（Whisper / Parakeet）跑在本地
- 录音文件存在本地
- AI 摘要默认用 Ollama（本地 LLM），也可以选 Claude/Groq/OpenAI——但数据由你决定是否出去

对于做医疗、法律、金融、国防的朋友来说，这简直是救命工具。

### 3. 跨平台

macOS（Intel + Apple Silicon）、Windows、Linux 全支持。Rust 后端 + Tauri 桌面壳，性能比 Electron 那堆吃内存的玩意儿强太多。

## 部署指南：3 分钟跑起来

### 方式一：直接下载（推荐）

去 [Releases 页面](https://github.com/Zackriya-Solutions/meeting-minutes/releases/latest) 下载对应平台的安装包：

- **Windows**: 下载 `x64-setup.exe`，双击安装
- **macOS**: 下载 `meetily_0.4.0_aarch64.dmg`，拖到 Applications
- **Linux**: 从源码构建（看下面）

### 方式二：从源码构建

如果你在 Linux 上，或者想自己魔改：

```bash
git clone https://github.com/Zackriya-Solutions/meeting-minutes
cd meeting-minutes/frontend
pnpm install
./build-gpu.sh
```

构建脚本会自动检测硬件加速：
- macOS: Metal + CoreML
- Windows/Linux: NVIDIA CUDA / AMD/Intel Vulkan

### 配置 Ollama 本地 AI 总结

为了让摘要也完全本地化，装个 Ollama：

```bash
# 安装 Ollama（macOS/Linux）
curl -fsSL https://ollama.com/install.sh | sh

# 拉一个不错的本地模型
ollama pull llama3.1:8b

# 或者用更轻量的
ollama pull qwen2:7b
```

然后在 Meetily 的 Settings 里选择 Ollama 作为 AI Provider，填上 `http://localhost:11434` 就行了。

## 实测体验

我跑了几天，说说真实感受：

### 转录准确率

- **英文**：用 Parakeet 模型，准确率大概 92-95%，对比 Otter.ai 的 95-97% 稍微差一点，但差距不大
- **中文**：用 Whisper 模型，准确率 85-90%，口音和专有名词偶尔翻车
- **实时性**：基本无延迟，说话后 1-2 秒出字

### 性能

M1 MacBook Air（8GB 内存）上，转录 + AI 总结同时跑：

- CPU 占用：30-45%
- 内存：~1.2GB
- 电池续航：大概掉电快 20%

还行。老一点的 Intel Mac 可能会吃力一些，建议用 Parakeet 小模型。

### 缺席功能

说实话，当前版本有一些明显的不足：

- **说话人识别（Speaker Diarization）**还在 Pro 版，社区版没有——这基本是刚需
- **没有日历集成**，不会自动加入会议
- **录音质量要求高**，如果是嘈杂环境，准确率直线下降

不过考虑到 MIT 协议 + 开源，这些都是可以自己贡献的。项目在快速迭代中，0.4.0 版本已经很能打了。

## 我的看法

**Meetily 不是 Otter.ai 的替代品，而是一种不同的哲学。**

如果你只看功能列表，它目前确实比不上商用的。但它的核心卖点不是功能，而是**主权**——你对数据拥有完整的主权。

对于个人开发者和小团队来说，它完全够用。对于大型企业，它的 Pro 版也在逐渐完善。

而且说实话，**2026 年的趋势就是 AI 往本地走**。从 Local LLM 到本地 RAG，从端侧模型到本地转录——云 AI 很好，但它不应该成为唯一选择。
